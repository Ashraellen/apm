## Naming rule for core project files

Core project files should use readable names first.

Abbreviations may remain in dictionaries or legacy notes, but should not be the main working file names when they make the structure harder to understand.

---

## Chapter file naming: storage vs index

Use two layers of chapter naming.

### Storage file name

For actual stored chapter files, use the compact stable format:

```text
E404GNFB1WM-LANG_CH-XXX_vN.ext
```

Example:

```text
E404GNFB1WM-RU_CH-001_v1.docx
```

This format is good for storage, sorting, multilingual versions, and version control.

### Index / table-of-contents name

In this folder index and in chapter maps, write a readable expanded line:

```text
E404_GNF_Book1_Why_Me_Glava_01_Oshibka_404_v1.docx — Глава 1. Ошибка 404
```

The readable line is a human navigation label, not necessarily the physical storage filename.

Rule:
Compact file names keep the archive clean. Readable index names keep the author sane.
