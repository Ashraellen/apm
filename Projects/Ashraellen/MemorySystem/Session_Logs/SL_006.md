## 1. Summary

This session clarified that book memory should not be stored in the global Ashraellen MemorySystem.

The user decided that each concrete book may later receive its own local MemorySystem inside that book's folder.

## 2. Accepted decisions

- The global `Projects/Ashraellen/MemorySystem/` remains a service layer.
- It should not store creative memory of books.
- Each concrete book can later have its own local `MemorySystem`.
- Book-local MemorySystem may store canon, characters, chapters, versions, translations and creative decisions for that specific book.
- This avoids mixing website, grants and book-writing layers.
