# Public Thoughts — правило навигации на страницах дуг

Дата фиксации: 2026-06-24  
Проект: Ashraellen MainSite / Ashraellen.com  
Связанный главный файл: `Projects/Ashraellen/MainSite/Decisions/Public_Thoughts_Multilingual_Rollout_Decisions_2026-06-23.md`

## Короткое правило

На самих страницах дуг опорных мыслей навигация должна быть навигацией именно по дугам, а не смешанной навигацией к витрине, архиву или отдельным мыслям.

Правильный вид нижней навигации на странице дуги:

```text
← Предыдущая дуга
Следующая дуга →
```

## Уровни навигации

Важно разделять уровни:

```text
Страница отдельной мысли
→ навигация по отдельным мыслям внутри дуги.

Страница дуги
→ навигация по дугам.
```

На странице отдельной мысли допустимы кнопки:

```text
← Предыдущая мысль
Следующая мысль →
Следующая дуга →  (только на последней мысли дуги)
```

На странице дуги не нужно смешивать уровни кнопками вроде:

```text
К новым мыслям →
К публичной витрине →
Старее →
Новее →
```

Вместо этого использовать только дуговую логику.

## Направление кнопок

Для читателя архив читается от более свежих дуг к более старым.

Поэтому:

```text
← Предыдущая дуга
→ ведёт к более свежей дуге / к живой витрине, если текущая страница дуги является самой свежей архивной дугой.

Следующая дуга →
→ ведёт к более старой дуге.
```

Пример текущей структуры после появления живой шестёрки 0013–0018:

```text
/[lang]/public/
→ живая витрина 0013–0018

/[lang]/public/thoughts/
→ ДУГА 0002: 0007–0012

/[lang]/public/thoughts/index-0001.html
→ ДУГА 0001: 0001–0006
```

Правильная навигация:

```text
/[lang]/public/thoughts/  (ДУГА 0002)
← Предыдущая дуга → /[lang]/public/
Следующая дуга → /[lang]/public/thoughts/index-0001.html

/[lang]/public/thoughts/index-0001.html  (ДУГА 0001)
← Предыдущая дуга → /[lang]/public/thoughts/
Следующая дуга → не выводится, потому что более старой дуги пока нет.
```

Если позже появится ДУГА 0003 как архивная страница, навигация должна обновляться по тому же принципу:

```text
ДУГА 0003
← Предыдущая дуга → /[lang]/public/  (если /public/ показывает живую ДУГУ 0004)
Следующая дуга → index-0002.html

ДУГА 0002
← Предыдущая дуга → /[lang]/public/thoughts/  (если там ДУГА 0003)
Следующая дуга → index-0001.html

ДУГА 0001
← Предыдущая дуга → index-0002.html
Следующая дуга → не выводится
```

## Языковые подписи

```text

```

```text

Next arc →
```

Polski:

```text
← Poprzedni łuk
Następny łuk →
```

## Практическое правило для будущих ротаций

При каждой ротации дуг проверять:

1. На страницах дуг карточки мыслей имеют видимую кнопку `Открыть мысль →` / локализованный аналог.
2. Нижняя навигация на странице дуги использует только дуговую навигацию.
3. Самая свежая архивная дуга ведёт кнопкой `← Предыдущая дуга` к живой витрине `/[lang]/public/`.
4. Кнопка `Следующая дуга →` ведёт к более старой дуге.
5. На самой старой дуге `Следующая дуга →` не выводится, если более старой дуги нет.

## Решение по текущему проходу

Исправить страницы дуг в уже сделанных языках:

```text
ru/public/thoughts/
ru/public/thoughts/index-0001.html

en/public/thoughts/
en/public/thoughts/index-0001.html

pl/public/thoughts/
pl/public/thoughts/index-0001.html
```

Дальше все новые языковые версии создавать сразу по этому правилу.
