## 1. Summary

This chat focused on adding the MONOLITH trilogy to the Ashraellen main website and bringing it into the existing MainSite visual and structural canon.

Main outcomes:

- Preserved `/monolith/` as a short public social-media hub / gateway.
- Created / refined language-specific MONOLITH portal pages.
- Added MONOLITH to the Books layer on all project languages.
- Created a proper Books-series architecture:
  - `/<lang>/books/`
  - `/<lang>/books/monolith/`
  - `/<lang>/books/monolith/beton/`
  - `/<lang>/books/monolith/sludge/`
- Added full Russian pages and later full multilingual volume pages for BETON and SLUDGE.
- Rewired the series pages so they no longer jump directly to Amazon; they now lead to internal volume pages, and the volume pages contain store links.

Accepted architectural formula:

```text
Books → MONOLITH → Volume → Store
```

Accepted public/social-media formula:

```text
/monolith/ → language detection → /<lang>/monolith/
```

## 2. Accepted decisions

### 2.1 Keep `/monolith/` as public hub

The user explicitly decided that:

- `https://www.ashraellen.com/monolith/` must remain a public hub because it has already been distributed through social networks.
- It should not become the detailed Books-series page.
- It should act as a fast multilingual portal / redirect gateway for Instagram, posts, descriptions, and other external references.

Accepted split:

```text
/monolith/                     — public social-media gateway / language hub
/<lang>/monolith/              — language-specific portal page
/<lang>/books/monolith/        — detailed book-series page
/<lang>/books/monolith/beton/  — detailed volume page
/<lang>/books/monolith/sludge/ — detailed volume page
```

### 2.2 MONOLITH belongs inside Books

MONOLITH is not only a standalone hub. It must appear in the site’s Books section as a proper book-series object.

Accepted route:

```text
/<lang>/books/
→ MONOLITH card
→ /<lang>/books/monolith/
→ /<lang>/books/monolith/beton/
→ /<lang>/books/monolith/sludge/
```

### 2.3 Series pages should not link directly to stores

Initial multilingual series pages linked book cards directly to Amazon. This was corrected.

Accepted correction:

- Cards for BETON and SLUDGE on `/<lang>/books/monolith/` must link to internal volume pages.
- Store buttons belong on the volume pages.
- This keeps the reader inside the site before sending them to Amazon / Google Play.

Final logic:

```text
Books → MONOLITH → Volume → Store
```

### 2.4 Portal pages can link directly to stores

The public portal `/monolith/` and language portals `/<lang>/monolith/` are intentionally short and functional.

Accepted behavior:

- They may show book cards with direct store buttons.
- They are not the detailed literary/Books pages.
- Their purpose is quick routing from social media.

### 2.5 Use existing site canon

Early MONOLITH design did not match the established site style. The user rejected that direction.

Accepted visual correction:

- MONOLITH pages must use the same internal-page logic as pages like `/ru/research/` and `/ru/books/you-are-already-online/`.
- Header pattern:
  - `Ashraellen` top-right.
  - small micro-menu under it.
- Footer/seal pattern:
  - small Ashraellen symbol / mark of presence.
- Cards should resemble existing `books.css` style: quiet, dark, translucent, rounded, not loud marketing blocks.

### 2.6 Covers and background file organization

Accepted asset organization:

```text
assets/backgrounds/monolith-bg.webp
assets/covers/beton-ru.webp
assets/covers/beton-en.webp
assets/covers/sludge-ru.webp
assets/covers/sludge-en.webp
```

Decision:

- Background images belong in `assets/backgrounds/`.
- Book covers belong in `assets/covers/`.
- The MONOLITH background path was corrected from:

```text

```

to:

```text

```

### 2.7 Languages

Project language set used in this work:

```text
en, ru, pl, de, es, fr, pt, uk
```

Accepted behavior:

- English and Russian currently have live published editions for available books.
- For PL / DE / ES / FR / PT, pages should exist now and link to English Amazon editions until local editions are available.
