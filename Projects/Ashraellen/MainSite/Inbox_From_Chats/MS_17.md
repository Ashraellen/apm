## 2. Accepted decisions

### 2.1. Site framing

The website must continue to present Ashraellen as:

```text
an independent multilingual literary-philosophical and artistic research practice
```

The site must not be primarily framed as:

```text
blog

spiritual teaching
therapy
motivation
self-help
infobusiness
personal-development content
request for personal rescue
```

Public-facing author name:

```text
Ashraellen
```

Formal / legal name when needed:

```text
Nikolai Kostyshev
```

Do not use `Ashraellen Live` as the current public author name unless quoting a specific legacy publication record.

### 2.2. Static HTML matters

Accepted technical principle:

```text
Core search / AI / preview data must exist in static HTML.
```

Do not rely on client-side JavaScript for:

```text
canonical links
core navigation
JSON-LD
Open Graph / Twitter Card tags
important semantic descriptions
```

### 2.3. PWA icons and Open Graph images are separate

The site can be installed as a PWA / app and already has dedicated app icons.

Decision:

```text
PWA icons must remain separate from Open Graph images.
```

Open Graph images should live under:

```text
assets/og/
```

They should be large social preview images, not small app icons.

### 2.4. Open Graph should initially use one strong default image

Decision:

```text
Use one strong default Open Graph image for now, then later create section/book-specific OG images.
```

The accepted default image is the dark formal Ashraellen visual with tie and hand-with-eye symbol.

### 2.5. Open Graph should cover all language pages, not only language entry pages

Initial OG scope was root plus language entries. User clarified that OG is needed on all language pages.

Accepted final scope:

```text
/
/en/...
/ru/...
/pl/...
/de/...
/es/...
/fr/...
/pt/...
/uk/...
/be/...
```

For `/books/` pages, `og:type` should be `book`; for other pages, `website` is acceptable.

### 2.6. Future asset/CSS consolidation should be planned, not rushed

User asked whether scattered backgrounds and per-page CSS should later be consolidated.

Accepted plan:

```text
Plan a future technical stage called Asset & Style Consolidation.
```

This should include:

```text
assets/backgrounds/
assets/styles/ or page-type CSS files
inventory before migration
no blind rewrite
```

Priority: Medium. It should not interrupt urgent grants, content or legal work.
