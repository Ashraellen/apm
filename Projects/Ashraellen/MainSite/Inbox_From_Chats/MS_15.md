The site is primarily for living visitors, readers and viewers. It must not look like a grant-application showcase.

Accepted positioning:

- the homepage stays visitor-oriented;
- grant/foundation logic should not dominate the front page;
- the professional layer exists for institutions, funds, publishers, residencies and patrons;
- professional information should be accessible but not visually aggressive.

### Professional dossier page

The professional page should remain named:

```text

```

```text

Грантовая заявка
```

Core idea:

> The professional dossier shows Ashraellen as a mature artistic-research project: author, method, works, public forms, development plan, infrastructure, expected outcomes, audience value, public response and cooperation.

### Old vs new page

The first test version was judged too thin compared with the old page. Decision:

- do not replace the old rich page with a dry short version;
- use the old density and add the new structure;
- keep strong blocks such as support, infrastructure, expected results, public response and audience value.

### CSS / HTML cleanup

Accepted technical decision:

- remove inline `<style>` from the professional HTML page;
- move styling to `assets/professional.css`;
- keep HTML clean;
- keep old page as `index-old.html` temporarily;
- delete intermediate test pages after the final page works.

### Development list styling

Accepted visual decision:

- in `План развития`, use strong round bullet points;
- avoid the old thin line/border-list look.

### In-page navigation

Accepted visual decision:

- compact text navigation at the top of the professional dossier;
- no large button-like navigation for in-page anchors;
- use small text links with dot separators.

### Public forms of the project

Accepted strategic wording:

> Ashraellen is not a set of separate YouTube channels. It is a system of public forms belonging to one artistic-research field.

This prevents the project from looking like a chaotic content farm and frames it as a coherent research ecosystem.

## 6. Technical notes

### GitHub tool limitation

The assistant verified GitHub repo permissions during the chat. Access to `Ashraellen/ashraellen` showed admin/push-level permissions, so the problem was not GitHub authorization.

Observed issue:

- some direct GitHub writes worked;
- some large HTML updates were blocked before reaching GitHub;
- even simple create-file attempts could sometimes be blocked by the OpenAI/tool safety layer;
- small updates to existing files sometimes worked.

Working rule established:

1. Try direct GitHub update for small existing-file changes.
2. For large HTML pages, prepare downloadable single `.html` files.
3. User uploads manually if connector blocks the write.
4. Keep CSS separate and clean.
5. Keep old versions or rely on Git history for safety.

### Analytics observation

The user saw traffic from the United States in analytics. Explanation given:

- assistant/web checks likely come from server/data-center infrastructure, not from the user’s Polish browser;
- US hits may partly be assistant checks;
- bots, Google/GitHub checks and previews can also appear;
- after active site work, analytics should be judged after 1–2 quieter weeks.

## 7. Positioning / grants

### Central grant positioning

Support should be framed as enabling sustained work on an already existing project, not as personal rescue.

Avoid:

```text
Grant will let me stop working as a taxi driver.
```

Prefer:

```text
Support will free sustained working time for the completion and public development of an already existing body of work.
```

### Current project seeking support

Important block restored and kept:

> Текущий этап Ashraellen требует устойчивого времени, технической базы и редакционной концентрации для превращения разрозненных материалов в целостную многоязычную художественно-исследовательскую платформу.

> Проект ищет поддержку для развития сайта Ashraellen.com, систематизации публичного архива, подготовки книг, перевода и адаптации материалов, производства видео- и аудиоформатов, а также создания ясной публичной структуры для читателей, зрителей, издателей, исследовательских партнёров и культурных институций.

> Поддержка нужна не для расширения ради расширения, а для перехода от фрагментарной работы к устойчивому циклу: наблюдение → текст → публикация → перевод → публичный отклик → архив.
