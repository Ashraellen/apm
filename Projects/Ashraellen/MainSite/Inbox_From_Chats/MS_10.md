## 2. Accepted decisions

### PWA / installable app

- Ashraellen.com should remain installable as a lightweight PWA / web app.
- It is not an App Store / Google Play app; it is the website installed through the browser with its own icon and standalone window.
- The correct `start_url` for the installed app is `/`, not `/?manual=1`.
- The manual language chooser should remain available at:

  `https://www.ashraellen.com/?manual=1`

  but it should not be the default entry point for the installed app.

- The installed app should open as:

  1. app icon;
  2. short splash/logo;
  3. direct redirect to the browser/system language version, for example `/ru/`.

- If Android/Chrome caches the old PWA state, the user may need to uninstall the app and clear site data for `ashraellen.com` / `www.ashraellen.com` in Chrome.

### PWA icon decisions

- The app icon should use the Ashraellen hand/eye symbol on a black background.
- The user preferred a version where the white contour/hand/eye elements were filled with a hammered/checkered silver-like effect, while not filling the entire interior.
- Final app icon was accepted visually by the user and installed successfully on Android.
- Separate favicon work may be done later; app icons and browser favicons should be treated as separate icon layers.

### Install instructions

- For users, the recommended wording is not “only Chrome” but:

  - Android: recommended Chrome; Edge / Brave / Samsung Internet may also work.
  - iPhone/iPad: Safari → Share → Add to Home Screen.
  - Desktop: Chrome / Edge install icon or browser menu.

- A future install button on the site is possible, but should not be rushed. Browser confirmation will always be required; it cannot install silently.
- Decision at this stage: do not add a site install button yet. PWA works; button can be a later interface improvement.

### Research method architecture

- The `/research/` page should remain a strong entrance / map of the research area, not be aggressively shortened.
- The core methodological explanation should live on a separate page:

  `/research/method/`

- The new method page should not be a placeholder; it must be a full page in every language.
- The Manifest of the Method should not become a separate page yet; it should remain inside the method page, hidden in a collapsible block opened by “Read / Читать manifesto”.
- The page `/research/method/` should explain why books, essays, short texts, video, visual symbols and audience response are parts of the research method, not just formats of content delivery.

### Language mirror / multilingual policy

- When a new structural page is added to the main multilingual site, prefer full multilingual implementation rather than adding exceptions to the language mirror check.
- Do not use placeholder pages for important conceptual sections.
- The strict language mirror check should remain strict; it protects site architecture.

## 4. Pages and structure

### Research section architecture

Current intended architecture:

- `/research/` — entrance / map: what is being researched, why, and where to go next.
- `/research/method/` — methodological foundation: how the work is done and why this is research, not merely content production.
- `/research/projects/` — large project models / research directions.
- `/public/posts/` — public short-form outputs: formulas, mini-essays, fragments, sources.
- `/research/archive/` — traces, versions, old cuts, process remains.

The accepted conceptual separation:

- Research page = “what and why”.
- Method page = “how and why it works”.
- Projects = “where the method unfolds”.
- Publications = “short cuts / public outputs”.
- Archive = “traces of process”.

### Method page internal structure

All language method pages follow the same HTML / content pattern:

- `h1`: Method of Observation / local equivalent.
- Intro note: phenomenology of everyday dimming/darkening of consciousness.
- Main explanatory sections.
- Collapsible Manifest block via `<details class="manifest">`.
- Back links to Research, Projects, Publications.
- Seal block with `assets/symbol.png`.

### Collapsible Manifest implementation

The Manifest is implemented with native HTML details/summary.

Relevant style pattern:

