## 2. Accepted decisions

- Keep automatic browser-language redirect on the root page, but root `index.html` must also contain static language links so the language menu remains usable without JavaScript.
- Root language fallback remains English if the browser language is unsupported.
- A root `404.html` page is needed and should be visually centered, without a border/frame.
- 404 layout decision:
  - large `404` in the center;
  - `not found` directly under the numbers, smaller;
  - phrase `The page has stepped out of form.` smaller;
  - language links styled like the second navigation menu on `/en/professional/`.
- For page-specific designs, create separate CSS files instead of inline `<style>` blocks.
- `SECCUS / АХЕПСУ` naming rule:

## 6. Technical notes

- GitHub Pages remains the deployment model.
- Root page should work with and without JavaScript.
- Root redirect script should not be the only way to access language pages.
- Use static language links in root navigation.
- Use separate page CSS files for distinctive page designs.
- SECCUS/AHEPSU page-specific CSS file:
  - `assets/books-seccus.css`
- The shared books CSS remains:
  - `assets/books.css`
- Correct CSS order:

```html
<link rel="stylesheet" href="../../../assets/books.css?v=...">
<link rel="stylesheet" href="../../../assets/books-seccus.css?v=...">
```

- Dedicated page class for SECCUS/AHEPSU:

```html
<div class="page book-seccus">
```

- Background rule in `books-seccus.css` uses:

```css
.page.book-seccus::before{
  background-image:url("backgrounds/seccus-bg.webp");
}
```

- Excerpt spacing was tuned in `books-seccus.css` to avoid overly large gaps:
  - `.excerpt { line-height: 1.46; }`
  - `.excerpt p { margin: 0 0 7px; }`
  - `.excerpt h4 { margin: 16px 0 8px; }`
