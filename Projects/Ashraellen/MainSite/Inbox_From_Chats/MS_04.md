## 6. Technical notes

### Namecheap DNS records configured

Expected active records:

```text
A Record     @      185.199.108.153
A Record     @      185.199.109.153
A Record     @      185.199.110.153
A Record     @      185.199.111.153
CNAME Record www    ashraellen.github.io.
```

The dot at the end of `ashraellen.github.io.` is acceptable. It marks the value as a fully qualified domain name and should not be manually fought.

### Old DNS state

Before correction, the domain used custom Beget nameservers:

```text
ns1.beget.co
ns1.beget.pro
ns2.beget.com
ns2.beget.pro
```

Those were replaced by Namecheap BasicDNS so DNS records could be managed directly in Namecheap.

### GitHub Pages state

GitHub Pages displayed:

```text
DNS check successful
```

for:

```text
www.ashraellen.com
```

However, `Enforce HTTPS` was still unavailable with the message that a certificate had not yet been issued for `www.ashraellen.com`.

### HTTPS / SSL issue

The HTTPS problem at the end of the chat was not DNS validation anymore, but certificate issuance delay / pending GitHub Pages SSL state.

Possible checks if it does not resolve:

- wait for GitHub Pages certificate issuance;
- refresh GitHub Pages settings;
- remove and re-add the custom domain only if GitHub remains stuck;
- check that no extra `A`, `AAAA`, `CNAME`, `URL Redirect`, Beget, or conflicting DNS records exist;
- check whether CAA records exist; if they do, GitHub/Let’s Encrypt must be allowed, usually with `letsencrypt.org`.

CAA explanation used in chat: CAA is a DNS record that tells which certificate authorities are allowed to issue SSL certificates for the domain. In most ordinary Namecheap setups there are no CAA records at all, which is usually fine.
