### Sitemap and robots

- Physical sitemap file:
  - `/sitemap.xml` at the root of the repository/site.
- Public expected URL:
  - `https://www.ashraellen.com/sitemap.xml`
- `robots.txt` exists in the root and contains:
  - `User-agent: *`
  - `Allow: /`
  - `Sitemap: https://www.ashraellen.com/sitemap.xml`
- Sitemap automation exists at:
  - `.github/workflows/sitemap.yml`
- Sitemap workflow triggers on push to `main` when these change:
  - `**/*.html`
  - `scripts/generate-sitemap.js`
  - `.github/workflows/sitemap.yml`
- It also supports manual trigger via `workflow_dispatch`.
- It runs:
  - `node scripts/generate-sitemap.js`
- If `sitemap.xml` changes, GitHub Actions commits it automatically with message:
  - `Auto-update sitemap`

### Sitemap generator logic

Script:

- `scripts/generate-sitemap.js`

Behavior:

- Recursively walks the repository.
- Includes `.html` files.
- Excludes:
  - `.git`
  - `.github`
  - `node_modules`
  - `assets`
  - `scripts`
  - `404.html`
  - Google verification HTML files matching `google*.html`
- Converts `index.html` to clean trailing-slash URLs.
- Uses `git log -1 --format=%cI -- "relativePath"` to set `<lastmod>` per HTML file.

Important nuance:

- CSS and background image updates alone do not trigger sitemap generation. This is correct because sitemap maps URLs, not visual assets.
- HTML edits do trigger sitemap generation.
