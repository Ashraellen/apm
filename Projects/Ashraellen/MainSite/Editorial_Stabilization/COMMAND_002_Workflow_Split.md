## Required change 1 — make sitemap workflow safe

Update:

```text
.github/workflows/sitemap.yml
```

The permanent automatic workflow should run on relevant pushes and manual dispatch, but only safe tasks.

It must keep:

```text
- checkout
- Node setup
- Audit page metadata: node scripts/audit-page-metadata.js
- Export page keyword workbench: node scripts/export-page-keyword-workbench.js
- Generate sitemap.xml: node scripts/generate-sitemap.js
- Submit sitemap URLs to IndexNow: node scripts/submit-indexnow.js
- Commit sitemap.xml and reports if changed
```

It must remove from the automatic push workflow:

```text
- node scripts/add-missing-keywords.js
- python3 scripts/clean-public-identity-head.py
- python3 scripts/repair-title-description-og.py
- python3 scripts/repair-structured-social-gaps.py
- python3 scripts/dedupe-page-metadata.py
- python3 scripts/apply-og-from-page-backgrounds.py
```

It must not commit HTML in the permanent automatic workflow.

The final `git add` in `sitemap.yml` must include only generated sitemap/report files, such as:

```text
sitemap.xml
reports/page-metadata-audit.md
reports/page-keyword-workbench.md
reports/page-keyword-workbench.json
```

Do not include:

```text
**/*.html
```

Do not include:

```text
reports/page-backgrounds.md
```

unless that report is still generated by a safe step in this workflow.

Recommended push path triggers for `sitemap.yml`:

```text
- '**/*.html'
- 'scripts/audit-page-metadata.js'
- 'scripts/export-page-keyword-workbench.js'
- 'scripts/generate-sitemap.js'
- 'scripts/submit-indexnow.js'
- 'b2cda9f3e8714a6db926f48c7251ea65.txt'
- '.github/workflows/sitemap.yml'
```

Do not let changes to emergency repair scripts trigger the permanent sitemap workflow unless there is a specific reason.

## Required change 2 — create manual emergency metadata repair workflow

Create or update:

```text
.github/workflows/metadata-repair.yml
```

It must be manual-only:

```yaml
on:
  workflow_dispatch:
```

It must require a confirmation input.

Recommended input:

```yaml
inputs:
  confirm:
    description: 'Type RUN_EMERGENCY_REPAIR to run broad metadata repair scripts'
    required: true
```

The repair job must run only if:

```yaml
if: github.event.inputs.confirm == 'RUN_EMERGENCY_REPAIR'
```

The manual emergency workflow may run the broad repair sequence:

```text
1. Add missing static keywords
2. Clean public identity in ordinary page metadata
3. Repair title description and OG metadata
4. Repair structured data and social gaps
5. Dedupe page metadata
6. Apply OG images from page backgrounds
7. Audit page metadata
8. Export page keyword workbench
9. Generate sitemap.xml
10. Submit sitemap URLs to IndexNow
11. Commit repaired HTML, sitemap.xml and reports if changed
```

The emergency workflow may commit:

```text
**/*.html
sitemap.xml
reports/page-metadata-audit.md
reports/page-keyword-workbench.md
reports/page-keyword-workbench.json
reports/page-backgrounds.md
```

This is allowed only in the manual emergency workflow.
