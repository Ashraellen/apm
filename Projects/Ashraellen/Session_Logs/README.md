# Session_Logs

Status: active project memory folder
Updated: 2026-06-22

Use this folder for short memory reports from future Ashraellen project chats.

## Purpose

Every substantial chat should leave a trace of what it changed or decided.

This prevents the project from losing decisions between chats.

## Filename rule

Use short numbered files:

`SL_001.md`, `SL_002.md`, `SL_003.md`.

Check this folder and choose the next free number.

If the folder cannot be checked, use `SL_XXX.md` temporarily.

## Required sections

Each session log should include:

```markdown
# SL_NNN — Session log

Date:
Project area:
Chat topic:
Status:

## 1. Summary
## 2. Accepted decisions
## 3. Done / created / updated
## 4. Files and paths
## 5. Commit SHAs
## 6. Open tasks
## 7. Rejected / outdated / uncertain
## 8. What to update in MASTER files
```
